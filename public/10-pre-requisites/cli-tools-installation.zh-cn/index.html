命令行工具

启动 AWS CloudShell

从 us-west-2 区域打开 AWS Cloud Shell。如果看到题为 “欢迎使用 AWS CloudShell “的弹出窗口，请单击关闭。

安装实用程序

在 cloudshell 上复制并粘贴以下命令，以安装将在后续实验中使用的先决实用程序（eksctl、kubectl 和 helm）。 Kubectl 是一个命令行工具，用于与 Kubernetes API 服务器通信。 eksctl 是一个简单的命令行工具，用于在亚马逊 EKS 上创建和管理 Kubernetes 集群。 Helm 是 Kubernetes 的软件包管理器，可帮助你在 Kubernetes 集群上安装和管理应用程序。

:::code{showCopyAction=true showLineNumbers=true language=shell} curl ‘:assetUrl{path="/code/pre-reqs.sh”}’ -output pre-reqs.sh –输出 pre-reqs.sh chmod 700 pre-reqs.sh ./pre-reqs.sh :::

:::alert{header=“Important” type=“warning”}

如果上述脚本输出 “This AWS CloudShell Environment is not in us-west-2 region."（此 AWS CloudShell 环境不在 us-west-2 区域），这意味着您不在us-west-2 AWS 区域。需要在us-west-2运行研讨会说明。单击AWS Cloud Shell在**us-west-2*启动 AWS Cloud Shell

:::
