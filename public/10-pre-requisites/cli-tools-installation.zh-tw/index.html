命令行工具

啟動 AWS CloudShell

從 us-west-2 區域開啟 AWS Cloud Shell。如果看到標題為 “Welcome to AWS CloudShell “的彈出視窗，請按一下 Close。

安裝公用程式

在 cloudshell 上複製並貼上下列指令，以安裝後續實驗中會用到的先決公用程式 (eksctl、kubectl 和 helm)。 Kubectl 是您用來與 Kubernetes API 伺服器通訊的命令列工具。 eksctl 是一個簡單的命令列工具，用於在 Amazon EKS 上建立和管理 Kubernetes 叢集。 Helm 是 Kubernetes 的套件管理員，可協助您在 Kubernetes 叢集上安裝和管理應用程式。

:::code{showCopyAction=true showLineNumbers=true language=shell} curl ‘:assetUrl{path="/code/pre-reqs.sh”}’ –output pre-reqs.sh chmod 700 pre-reqs.sh ./pre-reqs.sh :::

:::alert{header=“Important” type=“warning”}

如果上述腳本的輸出結果顯示「此 AWS CloudShell 環境不在 us-west-2 區域」，這表示您不在us-west-2 AWS 區域。需要在 us-west-2 執行研討會指示。按一下 AWS Cloud Shell 以在us-west-2啟動 AWS Cloud Shell

:::

注意 在執行此工作坊時，每次在 CloudShell 中複製和貼上指令時，您都會看到安全貼上警告，如下圖所示。選擇 Paste 執行指令。
