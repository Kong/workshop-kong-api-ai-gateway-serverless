コマンドラインユーティリティ

AWS CloudShellの起動

us-west-2 リージョンから AWS Cloud Shell を開きます。“Welcome to AWS CloudShell” というポップアップが表示されたら、Close をクリックします。

ユーティリティのインストール

Cloudshell 上で以下のコマンドをコピーして貼り付け、以降のラボで使用するために必要なユーティリティ (eksctl、kubectl、helm) をインストールします。Kubectl は、Kubernetes API サーバーと通信するために使用するコマンドラインツールです。eksctl は、Amazon EKS 上で Kubernetes クラスタを作成および管理するためのシンプルなコマンドラインツールです。Helm は Kubernetes 用のパッケージマネージャーで、Kubernetes クラスタへのアプリケーションのインストールと管理を支援します。

:::code{showCopyAction=true showLineNumbers=true language=shell} curl ‘:assetUrl{path="/code/pre-reqs.sh"}’ –output pre-reqs.sh chmod 700 pre-reqs.sh ./pre-reqs.sh :::

:::alert{header=“重要” type=“warning”}

上記のスクリプトで “This AWS CloudShell Environment is not in us-west-2 region.” と出力された場合は、us-west-2 リージョンではないことを意味します。us-west-2 でワークショップの手順を実行する必要があります。AWS CloudShell をクリックすると、us-west-2 で AWS CloudShell が起動します。

:::

注意: このワークショップを実施している間、CloudShell でコマンドをコピー＆ペーストするたびに、次の図のような警告が表示されます。Paste を選択してコマンドを実行します。
